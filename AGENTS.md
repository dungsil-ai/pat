# AGENTS.md

이 문서는 **코드 스캔만으로 파악하기 어려운 운영 지침**만 간결하게 정리합니다.

## 1) 문서 유지 원칙
- 코드/폴더 구조/세부 아키텍처처럼 저장소를 스캔하면 확인 가능한 설명은 AGENTS.md에 장문으로 반복하지 않습니다.
- AGENTS.md에는 팀이 항상 지켜야 할 **정책, 워크플로 규칙, 작성 규칙**만 유지합니다.

## 2) 필수 검증 규칙 (항상 적용)
- 모든 변경사항은 커밋 전에 **테스트 + 린트 성격 검사**를 반드시 수행합니다.
- 기본 검증 명령:
  - `pnpm test`
  - `pnpm exec tsc --noEmit` (별도 lint 스크립트가 없을 때 타입 검사로 대체)
- 실패 시 원인 해결 후 재실행하고, 성공 확인 뒤 커밋합니다.

## 3) 작성 언어 규칙
- 커밋 메시지, 문서, 코드 주석, 테스트 이름은 모두 한국어로 작성합니다.

## 4) 문서 반영 규칙
- README에는 일회성/세션 전용 정보(임시 우회, 로컬 실험 메모 등)를 절대 커밋하지 않습니다.
- 앞으로도 계속 지켜야 하는 운영 규칙이 생기면 README가 아닌 **AGENTS.md**에 추가합니다.

## 5) 작업 범위 규칙
- `upstream/` 산출물처럼 저장소에 추적되지 않는 외부 동기화 결과는 지침 문서의 핵심 설명 대상으로 삼지 않습니다.
- 실제 구현/명령/동작 세부는 코드와 스크립트를 단일 진실 원천(SSOT)으로 간주합니다.

## 6) 최소 명령 참고
- 테스트: `pnpm test`
- 타입 검사(린트 대체): `pnpm exec tsc --noEmit`
- 사전 정렬: `pnpm sort-dict`
- 번역 실행: `pnpm ck3`, `pnpm vic3`, `pnpm stellaris`

---
필요 시 이 문서는 계속 축약/정리하며, 코드에서 자동으로 검증 가능한 내용은 최대한 제외합니다.
